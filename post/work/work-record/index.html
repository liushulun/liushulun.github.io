微信一面（技术）

2021/07/22，总时长约 50 分钟。主要考察 Android 基础知识体系，但有一定深度，比较考验对知识的理解和融会贯通。

（1）技术题

  • Android 有哪些跨进程通信方式，以及它们之间有什么区别？
  • 简述 Binder 的原理
  • 既然 Binder 基于内存映射，那 Binder 如何保障进程数据的安全性？
  • 从一个 Activity 跳转到另一个 Activity，它们的生命周期流程是什么样的？
    • 需要考虑到 Activity 的启动模式、是否复用实例、Activity 的 Window 样式、等等情况，尽可能作答全面。
  • 为什么会发生 ANR，如何监控线上 ANR 情况？
  • 如何自己设计一个 ANR 监控工具？
  • 为什么会发生 OOM，如何监控线上 OOM 情况？
    • 大内存分配、内存泄漏
  • 什么情况下会发生内存泄漏，以及如何避免？
  • APK 打包流程
  • Android 消息处理机制
  • 为什么 Android 有异步消息、屏障消息，主要用途是什么？
    • 绘制渲染与 VSync
  • SurfaceView 的原理
  • GLSurfaceView 和 SurfaceView 的区别

微信二面（技术）

2021/07/29，总时长约 50 分钟。主要考察项目内容，比较考验对项目中的技术方案的思考深度。

（1）技术题

  • SurfaceView 和 TextureView 的区别
  • SurfaceView 和 TextureView 哪个更耗电，为什么？
  • （针对项目）ARouter 框架的理解
  • （针对项目）Redux 框架的理解
  • （针对项目）为什么 ARouter 和 Redux 是中心化框架？
  • （针对项目）你提到的「微中心」模型与 ARouter、Redux 相比优势在哪
  • （针对项目）微中心模型的设计思路是什么
  • （针对项目）进房秒开优化做了哪些事情？
  • （针对项目）播放器实例复用主要做了什么？
  • （针对项目）空指针 Crash 优化主要做了什么，优化效果有数据支撑吗？

TikTok一面（技术）

2021/07/08，总时长约 60 分钟。非常简单，另一位朋友一面也同样很简单，可能字节的面试风格倾向于一面只了解基本情况。

（1）技术题

  • Android 消息处理机制（Handler、Looper、MessageQueue、Message、ThreadLocal）
  • APK 编译打包流程
  • （针对项目）Crash 优化怎么做的
  • （针对项目）直播中台框架搭建思路
  • （针对项目）直播进房秒开优化怎么做的
  • （针对项目）你上面说的这些，有上线后的数据效果吗

（2）编程题

  • 给你一个二维整形数组，并且已知每一行从左到右都是递增的，每一列从上到下也是递增的，然后给定一个目标数 target，从二维数组中找出是否包含该目标数，是否有时间复杂度小于 O(n^2) 的方法？
    • 整体思路：按照从大到小选择行 / 列的方式排除首位就大于 target 的行 / 列

TikTok二面（技术）

2021/07/14，总时长约 50 分钟。技术偏向于 Android 基础，回答不是很好，和一面难度跨度比较大。

（1）技术题

  • 知道 NoClassDefoundError 错误吗，什么情况下会抛出这个错误
  • ClassNotFoundException 和 NoClassDefoundError 的区别是什么
  • 假如一个 class 已经打包进 APK 了，是否还有可能在加载类时抛出 ClassNotFound 异常
  • Android 5 开始开启了 ODex 优化，那还会因为 MultiDex 导致 App 启动时抛出 ClassNotFound 吗
  • SurfaceView 和 TextureView 的区别
  • 假如要统计从 startActivity 开始到 Activity 真正被渲染首帧为止的耗时，会在哪两个地方埋点
  • 除了 View#post(Runnable) 之外，还有什么办法判断 Activity 渲染出了首帧
    • VSync, OnWindowFocusChanged
  • 为什么 View#post(Runnable) 可以确保在渲染完成之后才回调
    • 从 Activity 启动流程、创建 Window、ViewRootImpl、Choreographer、VSync、scheduleTraversal 角度回答
  • （针对项目）弹幕是怎么实现的，为什么要选择 SurfaceView
  • （针对项目）如果弹幕太多了会怎么处理来提升用户体验，有淘汰策略吗

（2）编程题

  • （20 分钟）给出一个无序整型数组、以及一个目标数 target；找出数组中一段连续的子数组，使得其中的元素之和等于 target；求满足这个要求的最短子数组长度。
    • 整体思路：滑动窗口 5 分钟写完，然后口述了优化思路，即先通过滑动窗口找到第一个符合的子数组，假设此时窗口宽度（即子数组长度）为 size，则接下来的遍历，窗口的宽度只需要从 size 到 0 即可，如果找到了更短的，则更新 size 的值。

微视二面（技术）

2020/03/14，总时长约 66 分钟。二面主要针对简历中写到的点问得很仔细，基本上项目经历中写到的每句话都问了相关知识点，还好自己的简历写得还是比较真实。我的简历中甚至还有一个错误在过去几个月的面试中都没有发现，这一次被指出来也是相当尴尬。总结出三点：简历可以包装但绝对不能过度吹嘘；项目经历中涉及到的技术点，或者说做项目时遇到的技术点，一定要尝试去深入了解、知其所以然，不然很容易一问就倒；面试过程一定要真诚，会就会、不会就不会，可以委婉表示或者尝试切入到自己会的内容，但切忌不懂装懂。

（1）面试题

  • （针对项目）介绍一下这个项目（简历中写了三个，都问到了）
  • （针对项目）为什么选择 Socket 进行数据传输？
  • （针对项目）申请了什么权限？如果用户没授权怎么办？
  • （针对项目）你都做了哪些用户体验的优化？
  • （针对项目）数据的缓存是怎么实现的？Glide 的缓存（项目中用的是 LRUCache）有了解过吗？
  • （针对项目）有没有做过一些性能指标的监控或者优化，比如启动速度、帧率等等？
  • （针对项目）数据传输的速度有多快？有没有导致 App 卡顿？如果数据量过大，有没有做一些优化（例如降采样等）？
  • （针对项目）用的数据协议是什么？方便透露一下协议的格式吗？
  • （针对项目）做了哪些数据埋点？
  • （针对项目）方便透露一下用户量吗？
  • （针对项目）如何优化的 RecyclerView 中的视频 Item？
  • （针对项目）多久更新一个版本？
  • （针对项目）异常上报和热更新（项目里用的是 Bugly 和 Tinker）是怎么实现的？Tinker 的原理有了解过吗？
  • （针对项目）项目中 FFmpeg 是怎么用的？
  • LRU 算法的理解
  • 在收到异常上报时，一般因为混淆过，可能收到的报告看不出来实际的地方，那你是如何解决的呢？
  • 项目中哪些情况下会出现 ANR，如果遇到 ANR 如何定位和修复呢？
  • 什么情况下会发生 OOM？如何定位和优化？
  • 有哪些方法可以监测 FPS？
  • LeakCanary 的原理有了解过吗？
  • 简述一下进行 JNI 开发的流程
  • 如果发生了 Native Crash，如何定位和修复？
  • 平时提交代码时，谁来 Review 以及如何 Review？
  • 有做过单元测试吗？是自己写得还是直接用的自带的？
  • Python 中的 nosetest 的原理？
  • 有没有遇到过比较难解决的报错或者 Crash（Application 不在主 dex 导致混淆后报 ClassNotFound）？最后又是怎么解决的？
  • 自己去设计一个线程池的话，会怎么考虑，例如怎么管理线程等
  • 平时写代码遵循的是：谷歌、阿里、或者是你自己的一套代码规范？
  • 平时玩游戏吗？假如你在休假时间在玩游戏，这时候领导打电话说项目出问题了，你会怎么解决？假如这个问题不是你负责的呢？假如你身边没有条件（例如没有电脑等）立即解决，又会如何处理？

    我个人觉得，在不考虑性能的情况下，DCL 和静态内部类的安全性是一致的所以两个都写了，如果考虑性能，静态内部类会更好一些。

华为

  • 岗位：消费者 BG - 音频产品开发部
  • 周期：2020/02/25 ~ 2020/03/23
  • 结果：已通过面试和提交材料，等待 Offer 发放流程。

整体感受：面试过程不好评论，面完第一轮才知道是外包，不过既然已经面了一次就索性坚持到底了。就像匆匆忙忙的面了又匆匆忙忙的过了（orz）。说实话交流效率还是有待提升，和 HR 联系时常收不到回复，可能是工作忙也可以理解。考察内容很全面，机试题、性格测试、面试都有，不过难度不大，面试过程和节奏也都比较轻松，也可能因为是外包的原因所以整体难度都下调了一些。

其实面试的是华为的高端外包 OD，据 HR 和面试官说，华为现在只有职级在 18 以上的高端岗位才社招，13-17 级的岗位不社招了，都是通过校招、OD、以及普通外包这三种方式招人。根据 HR、面试官、某位华为正式员工朋友的说明以及在网上收集到的信息来看，OD 和普通外包的一些信息和区别如下：

普通外包就是类似于：中软、软通、佰钧成、东软、文思海辉等，合同一般一年一签，人事制度由外包公司管理，有时工作地点不在华为，不接触核心业务，工号 WX（外协）开头，员工权限和工资低于正式员工，无奖金、无期权。

OD 和德科签合同，四年一签，人事制度由华为管理，工作地点在华为，接触核心业务，工号 3 开头（专门开设的），员工权限、工资、奖金和正式员工一致（据说实际上还是比正式员工差，也可以理解），但没有期权，要求学历一本及以上，每年有 20% 的机会可以转为正式员工。

华为一面（技术）

2020/02/28，总时长约 40 分钟。有两个面试官，都挺随和的，问题比较基础，有些不太清楚的也会给些提示引导我去思考，整体氛围比较 Nice，不会感到紧张。手写程序题是面完结束后才写的，然后拍照邮件发给 HR 再审核。

（1）面试题

  • Handler 的原理
  • HashMap 的原理
  • TCP 三次握手和四次挥手中间的字段及其含义（SYN、ACK、seq、ack 等）
  • 设计模式（单例、工厂、中介者等）
  • 应用架构（MVC、MVP、MVVM）
  • 讲一下你对 Activity 的理解
  • Android 触摸事件的传递
  • 用过反射吗？怎么理解？
  • （针对项目）介绍一下 App，以及项目中你的职责？

（2）编程题

  • 给出一个 32 位的有符号整数，将每一位的数字反转，如果反转后的整数溢出，则输出 0。例如：输入 123 则输出 321。
  • 编写一个函数来查找字符串数组中的最长公共前缀，如果不存在则返回空字符串。

华为二面（主管）

2020/03/11，总时长约 10 分钟。本来约在下午 15:00，提前了 20 分钟左右进入视频面试页面等待，正好 HR 联系说主管等会有事，让提前开始，就直接开始面了，面试过程也很短暂，没问什么技术细节，10 分钟左右面完，是我毕业以来面得最快的一次，不知道是不是因为是外包岗位的原因。不过面试官多次提到：“其实按照你的项目经验和毕业学校来说，你可能只能去普通外包”，算是稍微感觉到了一些不友好，不过说到底确实自身实力不够硬也怨不得别人，面试官毕竟是要为公司负责的可以理解。

（1）面试题

  • （针对项目）介绍一下 App
  • 给我一个理由说服我，为什么要把你招进高端外包而不是普通外包？

（2）编程题

无。

  • 结果：收到 Offer，因为有更心仪的所以放弃了。

数码视讯

  • 岗位：深圳研究院一所 - Android 开发工程师
  • 周期：2019/11/07 ~ 2019/12/18
  • 结果：拿到 Offer，但最终放弃入职

整体感受：第一次有被重视和珍惜的感觉。初面其实并不顺利，但二面三面和面试官都很聊得来，算是重新挽回甚至逆袭了吧。比较遗憾的是，即将入职时才发现和一开始了解到的偏差很大，有些东西一开始没有互相了解清楚，最终耽误了双方的时间，结局不算愉快。其实拿到 Offer 后，入职体检、预习项目等等都认真去做了，是真心抱着要去发展的心态入职的，连周边的房租都了解了一圈，公司为了入职的流程也做了很多工作，HR 也帮我申请了一次加薪机会而且也确实批下来了，确实对最后没能愉快合作感到抱歉。不过话说回来，我个人还是认为选择大于努力，如果发现道路、方向不对，一定要有决心朝着自己的目标走，毕竟有时候一个选择的失误可能会导致很多不可挽回的后果。

白骑士

  • 岗位：SDK 开发部 - Android 开发工程师
  • 周期：2019/10/12 ~ 2019/10/12
  • 结果：两次一面挂

整体感受：对这家公司的评价只有两个字：无语。

由于毕业后急于找工作，没有好好准备就入职了第一家公司，所以对于面试经验积累得比较晚，简历也是后来才完善起来。因此在开始找工作的开始一段时间收到的邀请都是比较差的，也一直没有拿到心仪的公司的邀请函。后来一次机会接到了白骑士的邀请前去面试，也 曾经 成为了我参与面试的公司里面最心仪的一家，甚至 愿意降薪 加入，不过很遗憾最终没有拿到 Offer。我参与了两次面试都挂了，第一次面完一面后面试官对我的评价挺高的（恕我愚昧是真的感觉到在赞同我），但是等了一周都没有消息，期间打前台电话咨询了两次，两次都说帮我登记了稍后回电告知，但直到现在我也没等到那个“稍后”的回电。一个星期后我觉得就是婉拒了吧，结果又接到另一位 HR 的邀约（内部系统也不同步信息？），我还是没放弃，猜测可能是第一次提的期望薪资太高了，所以又带着一丝希望前往面试，这次期望薪资少说了 2K（直接说的岗位邀约薪资范围的最低数），笔试和一面又是很顺利的通过了，结果回去以后又是杳无音讯，期间打过一次电话询问结果，和第一次一样还是到现在都没等到那通回电。

我个人是很反感这种行为的，如果技术不过关或者期望薪资太高，完全可以直接和我说，如果是让回去等通知婉拒，也没必要都主动打电话问了还这么敷衍。

同行者科技

  • 岗位：应用开发部 - Android 开发工程师
  • 周期：2019/10/08 ~ 2019/10/08
  • 结果：二面挂

整体感受：可以说是我后来面试的一个转折点，所以记忆尤其深刻。为什么说这次面试是转折点呢，因为深圳其实小公司是非常非常多的，所以对于我这样一个毕业一年的人，有很多小公司是从学历上挑（虽然其实并不高）邀请去面试。而在面试同行者之前，虽然面试经验、简历含金量、知识水平一直都很差，但毕竟面的是小公司所以基本上只有工资谈不拢没有直接拒绝的，这导致我一方面自我感觉良好觉得 Offer 很好拿，另一方面又很郁闷没有大 Offer。而这次面试真正打击了我、完全撕开了我的舒适区，见识到了真正的面试应该具备怎样的能力以及自己懂的到底有多贫乏，至此之后我开始一遍又一遍重写简历、总结面试、梳理知识点等等，直到现在完善出了一套自己的面试风格、也不再对面试感到紧张、甚至有时候还能反过来控制面试节奏，都要感谢这次面试带给我的挫折。一句话自勉一下：技术这条路崎岖坎坷，但我的征途，是星辰大海！

同行者一面（技术）

去面试报到后，同行的应聘人是相当多，光是 Android 都同时有好几位竞争者，每个人都发了一份 Java 编程题，需要在一个小时内手写完成一共三道题。现在回想题目并不难，不过由于当时刚计划离职开始面试，没有准备也没有经验所以做得很一般。做完后等了一会被叫进去面试，一面的面试官直接对笔试题，因为当时水平是真的不行，有很多边界值、限定范围等等的没有考虑到，还有一些小的逻辑错误，所以批改得惨不忍睹。不过一面的面试官很有耐心，一直在和我分析、带我反思，批改完也没问别的让我好好改改，然后去叫二面的面试官。

被安排在一个桌子上写得还有另一个年纪相仿的同行，一直在偷看我的答案，明着挡暗着挡好几回了也不收敛，相当反感这种作弊行为。尤其是明明我自己做的也很一般，偷看我的答案几乎一点用都没有……

同行者二面（技术）

二面面试官进来后也是直接对了一下笔试题，按照我改过的答案重新问了一遍，然后粗略的问了一些 Android 视图方面的问题，然后让我静待结果。没多久就回来对我说，很抱歉觉得我可能不是很适合他们公司。也在意料之中，我表示“没关系，也感谢贵公司邀约”后就回去了。

天地飞

  • 岗位：开发部 - Android 开发工程师
  • 周期：2018/08/20 ~ 2018/08/24
  • 结果：入职
  • 工作：2018/08/27 ~ 2019/12/05

整体感受：人生中的第一份工作，虽然工作过程中有很多的不如意，但不论怎么说毕竟是第一份工作、是毕业后第一个舞台，所以还是心存感恩的。天地飞的开发部只有一个懂 Java 的人，因此也就只有一个懂 Android 的人，我入职就接了这个人的班。不过一个人负责这个岗位对于应届毕业生来说还是很有好处的，因为本科期间在学校做的东西对于工作后的项目来说都太小儿科了，而工作项目有很明确的完成指标，这就被迫要自己从 0 开始学很多东西。所以在天地飞的工作经验虽然没有做出含金量很高的项目，但却是极大地锻炼了我的个人能力，从这个角度出发这份工作是值得的，毕竟锻炼的机会比工资对于一个新人来说更有用。而现在我也已经离职了，算是和公司友好分手，离职前也是尽心尽力把最后一份工作、最后一行代码、以及交接材料认真做好、测试无误才走的，就祝愿未来的我和天地飞都能共同成长共同进步吧！

天地飞一面

面试基本没有什么过程可言，面试官是我入职后的直属 Leader，因为工作方向不一样所以没有问什么技术问题，基本上就是聊了一下公司的情况和岗位职责就谈了工资。由于在本科期间错过了所有的校招，加上毕业后在广州和深圳找工作也不太顺利，甚至差点就参加培训机构了，所以对于找工作有点过于心急。当时收到天地飞的面试邀请后，甚至直接定了最近一班的火车赶过去面试，为了省下开酒店的钱半个小时不到面试完又赶回来，入职后也直接坐凌晨四点多的火车赶去上班。当时听到给的工资还是感觉不错的，但是因为没有经验，入职后才发现不符合预期，所以也一直感觉有些遗憾。
